# Debugging Log - July 14, 2026 (Crawler Service)

This log documents two critical issues identified and resolved during crawler deployment and live testing.

---

## Issue 1: Headless Browser Crash in Docker Container
- **Symptom**: The `chatbot_app` container crashed during crawl execution with:
  ```
  Looks like you launched a headed browser without having a XServer running.
  Set either 'headless: true' or use 'xvfb-run' before running Playwright.
  ```
- **Root Cause**: The application code (`app/crawl/service.py`) dynamically loaded `MODE=DEBUG` from the dev environment and set `headless=False`. In a containerized Linux environment lacking an X11 server, headed launches crash immediately.
- **Resolution**:
  1. Reverted [service.py](file:///c:/Users/akliv/Desktop/AkeshPersonal/ChatBot/app/crawl/service.py#L97-L100) to always default to `headless=True`.
  2. Moved the dynamic browser popup logic to [test_crawler.py](file:///c:/Users/akliv/Desktop/AkeshPersonal/ChatBot/tests/crawl/test_crawler.py#L28-L43) by patching `PlaywrightScraper` with a wrapper class (`CustomPlaywrightScraper`) that forces `headless=False` exclusively during local manual test suite runs.

## Issue 2: Repetitive Crawls & Duplicate HTML Storage Files
- **Symptom**: The crawler saved duplicate files for the same target page (e.g. multiple `apply-now` files with differing MD5 hashes: `apply-now_064e1085.html`, `apply-now_b3b59449.html`).
- **Root Cause**: The link extractor only stripped fragment IDs (`#...`). It treated URL variations created by trailing slashes (`/apply-now` vs `/apply-now/`) and tracking query parameters (e.g., `?utm_source=...`, `?gclid=...`) as distinct URLs, causing the crawler to enqueue and scrape the same document multiple times.
- **Resolution**:
  1. Created `LinkExtractor.normalize_url(url)` in [utils.py](file:///c:/Users/akliv/Desktop/AkeshPersonal/ChatBot/app/crawl/engine/utils.py#L17-L58):
     - Strips trailing slashes for standard page paths.
     - Discards analytics tracking query parameters (`utm_*`, `gclid`, `fbclid`, `sessionid`, `sid`, etc.).
     - Sorts any remaining query parameters alphabetically.
  2. Applied URL normalization to the scraper's base URL, outbound link discovery lists, and seed URLs in [crawler.py](file:///c:/Users/akliv/Desktop/AkeshPersonal/ChatBot/app/crawl/engine/crawler.py#L57-L68) to ensure only clean, unique URLs are tracked and visited.

## Issue 3: Concurrent Database Session Transaction Rollback (Integrity Error)
- **Symptom**: During parallel crawls, concurrent worker tasks crashed with the database exception:
  ```
  This Session's transaction has been rolled back due to a previous exception during flush. 
  To begin a new transaction with this Session, first issue Session.rollback(). 
  Original exception was: UniqueViolationError: duplicate key value violates unique constraint "crawled_pages_pkey"
  DETAIL: Key (id)=(61) already exists.
  ```
- **Root Cause**:
  1. A single shared database session (`AsyncSession`) was created at task startup and reused across all concurrent workers. SQLAlchemy sessions are not coroutine-safe or safe for parallel database queries.
  2. When a query in one worker raised an `IntegrityError` (e.g. duplicate key or sequence overlap), the shared session was poisoned and rolled back. This immediately caused all other parallel worker database queries to throw rollback failures.
- **Resolution**:
  1. Updated `CrawlStorageManager` in [storage.py](file:///c:/Users/akliv/Desktop/AkeshPersonal/ChatBot/app/crawl/engine/storage.py#L129-L135) to accept `db=None` in concurrent loop context. If `db` is `None`, the storage manager dynamically opens a short-lived `SessionLocal()` session for each isolated save operation, committing and closing it immediately.
  2. Updated page success and failure counter increments in [crawler.py](file:///c:/Users/akliv/Desktop/AkeshPersonal/ChatBot/app/crawl/engine/crawler.py#L245-L255) to instantiate their own database sessions dynamically if `db` is `None`.
  3. Re-configured [service.py](file:///c:/Users/akliv/Desktop/AkeshPersonal/ChatBot/app/crawl/service.py#L108-L113) to pass `db=None` to the storage manager in production, preserving mock session injection compatibility for test suite runs.
